## 1. 产品概述

裁缝店/改衣店管理系统，用于管理衣物从收件登记到完工取件的完整业务流程。
- 解决传统裁缝店纸质记录混乱、尺寸追踪困难、定价不透明、取件通知不便等痛点
- 目标用户为裁缝店店主、改衣技师及前台接待人员

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|---------|---------|
| 店员 | 系统内置账号 | 收件登记、测量录入、定价计算、状态管理、取件通知 |

### 2.2 功能模块
1. **订单列表页**：订单概览、状态筛选、快速搜索、新增订单入口
2. **订单详情/编辑页**：衣物信息登记、尺寸测量、修改项目定价、状态流转、瑕疵确认
3. **取件通知页**：待取件列表、一键通知、取件确认
4. **价格配置页**：修改类型单价配置、品类管理

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 订单列表页 | 顶部统计卡片 | 显示待处理/制作中/待取件/已完成订单数量 |
| 订单列表页 | 订单筛选区 | 按状态筛选、关键字搜索（客户姓名/电话/订单号） |
| 订单列表页 | 订单列表 | 卡片式展示订单摘要，支持快速查看/编辑 |
| 订单登记页 | 客户信息 | 客户姓名、联系电话、取件日期 |
| 订单登记页 | 衣物信息 | 品类、面料、品牌、颜色、修改需求描述 |
| 订单登记页 | 尺寸测量 | 关键部位原尺寸与目标尺寸对照录入 |
| 订单登记页 | 修改项目 | 选择修改类型（改短/放码/收腰/换拉链/补洞/改领口），单价自动累加 |
| 订单登记页 | 瑕疵确认 | 收件时衣物原有瑕疵记录及双方确认 |
| 订单状态页 | 状态流转 | 待处理→制作中→待取件→已完成 |
| 订单状态页 | 完工通知 | 点击发送取件提醒给客户 |
| 取件确认页 | 取件签收 | 确认款项结清、衣物完好交付 |
| 价格配置页 | 修改类型管理 | 修改类型名称、单价维护 |

## 3. 核心流程

客户携带衣物到店 → 店员登记客户信息与衣物详情 → 录入关键部位尺寸 → 选择修改项目自动计算总价 → 记录并确认衣物原有瑕疵 → 订单进入制作状态 → 完工后更新状态并通知客户取件 → 客户到店取件确认交付。

```mermaid
flowchart LR
    A["客户到店"] --> B["登记客户与衣物信息"]
    B --> C["录入原尺寸与目标尺寸"]
    C --> D["选择修改项目计算价格"]
    D --> E["确认衣物原有瑕疵"]
    E --> F["订单进入制作中"]
    F --> G["完工标记待取件"]
    G --> H["发送取件通知"]
    H --> I["客户取件确认交付"]
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深咖啡色 #5D4037 搭配暖米色 #F5F0E8，营造传统裁缝店的匠心质感
- 辅助色：金色 #C9A962 作为强调色（按钮、徽章）
- 按钮风格：圆角矩形，轻微阴影，悬停时上浮效果
- 字体：标题使用 Noto Serif SC（衬线体，经典稳重），正文使用 Noto Sans SC（清晰易读）
- 布局风格：左侧导航栏 + 右侧主内容区，卡片式模块布局
- 图标风格：线性风格图标，统一 24px 尺寸，与主题色协调

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 订单列表页 | 统计卡片 | 背景渐变、大号数字、状态色标、悬停微动效 |
| 订单列表页 | 筛选栏 | 标签式状态切换、圆角搜索框、主色按钮 |
| 订单列表页 | 订单卡片 | 客户姓名、衣物信息、状态徽章、操作按钮 |
| 订单登记页 | 表单区域 | 分区标题、输入框组合、尺寸对照表格、价格汇总面板 |
| 订单登记页 | 瑕疵确认 | 复选框 + 签名区样式、确认按钮 |
| 价格配置页 | 项目列表 | 表格展示、行内编辑、快速修改 |

### 4.3 响应式
桌面端优先设计，适配 1280px 及以上屏幕。平板端自动调整卡片布局为单列，移动端折叠导航为底部 Tab 栏。
